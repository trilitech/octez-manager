# What is Octez Manager?

Octez Manager is a CLI and TUI tool for deploying and managing Tezos infrastructure.

## The problem

Setting up Tezos infrastructure requires:

- Writing systemd unit files
- Managing environment configurations
- Handling user permissions
- Downloading and importing snapshots
- Monitoring logs across services

## The solution

One command:

```bash
octez-manager install-node --instance mainnet --network mainnet
```

Or an interactive TUI:

```
┌─ Install Node ───────────────────────────────────────────────────────┐
│                                                                      │
│  Instance name:     mainnet                                          │
│  Network:           mainnet                                          │
│  History mode:      ● rolling  ○ full  ○ archive                     │
│  Bootstrap:         ● snapshot  ○ genesis                            │
│                                                                      │
│                              [ Install ]                             │
└──────────────────────────────────────────────────────────────────────┘
```

## Services

| Service | Description |
|---------|-------------|
| **Node** | L1 node with automatic snapshot bootstrap |
| **DAL Node** | Data Availability Layer |
| **Baker** | Block production and attestations |
| **Accuser** | Double-baking/endorsing detection |

## Modes

**User mode** — Services run as your user. Good for testing.

```bash
octez-manager ui
```

**System mode** — Dedicated service users, proper isolation. For production.

```bash
sudo octez-manager ui
```
